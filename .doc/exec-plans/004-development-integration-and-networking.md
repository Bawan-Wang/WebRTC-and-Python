# Development Integration and Networking 執行計畫

## 目的

讓 Vue 前端與 Python 後端在本機與同網段環境下可以穩定整合，並為後續 STUN、HTTPS 與部署能力預留結構。

## 範圍

### 本次範圍內

- Vite dev server proxy
- API base URL 與環境變數規劃
- 本機 CORS 與開發整合
- 單機與同網段測試路徑
- STUN 與 HTTPS 需求評估

### 本次範圍外

- 正式 TURN 服務部署
- 雲端基礎設施自動化
- 反向代理與正式憑證自動化

## 目標產出

- 一組可供前後端本機協作的開發設定
- 一組明確的環境變數命名與使用方式
- 一份網路條件與限制說明

## 執行項目

### 1. 本機開發整合

- 設定 Vite proxy 指向 Python API
- 規劃前後端埠號與啟動方式
- 處理 CORS 或避免跨來源問題

### 2. 環境變數與設定模型

- 定義前端 API base URL
- 定義 STUN server 設定的注入方式
- 定義後端 host、port 與 debug 設定
- 為未來 HTTPS / TURN 預留設定欄位

### 3. 網路條件驗證

- 驗證 localhost 開發模式
- 驗證同網段不同裝置間的連線可行性
- 盤點沒有 STUN/TURN 時的限制與錯誤表現

### 4. 部署前注意事項

- 記錄 `getUserMedia()` 的 HTTPS 要求
- 記錄 STUN/TURN 的適用時機
- 記錄企業網路、NAT 或防火牆可能造成的風險

## 驗證方式

- 前端在開發模式可不修改硬編碼 URL 即連到後端
- 本機與同網段能完成最小送流測試
- 設定檔可清楚區分開發與部署需求

## 完成定義

- 新加入的開發者可依文件啟動前後端
- 前後端整合不需要手動修改多處設定
- 後續若補 STUN/HTTPS 只需在既有設定模型中擴充

## 風險與注意事項

- 若未先釐清本機與部署場景，後續配置會反覆修改
- 缺少 HTTPS 與 ICE server 規劃時，容易在跨裝置測試階段卡住
- Vite proxy 只能解決開發期問題，不能替代正式部署架構
