# Validation and Documentation 執行計畫

## 目的

將整體遷移工作的驗證方式、手動測試路徑、故障排查與使用說明整理成可重現的文件，避免功能可用但無法交接。

## 範圍

### 本次範圍內

- 驗證案例整理
- 手動測試步驟
- 常見錯誤與排查方式
- README 與架構文件更新
- Demo 操作說明

### 本次範圍外

- 大規模自動化 E2E 測試平台
- 效能基準測試平台
- 完整產品級監控儀表板

## 目標產出

- 一份可照著操作完成驗證的測試清單
- 一份更新後的 README
- 一份常見問題與故障排查指引

## 執行項目

### 1. 手動驗證案例

- 驗證前端能取得攝影機權限並顯示預覽
- 驗證前端可成功送出 offer 並套用 answer
- 驗證後端可收到 track 並處理 frame
- 驗證停止送流、重整頁面與中斷連線時的清理

### 2. 錯誤情境驗證

- 瀏覽器拒絕攝影機權限
- 後端未啟動或 signaling API 錯誤
- 無 STUN/TURN 時跨網路連線失敗
- 後端處理迴圈拋出例外

### 3. 文件更新

- 更新 README 的架構與啟動流程
- 更新開發環境設定說明
- 補上埠號、環境變數與已知限制
- 將 `architecture.md` 與 exec plan 關係說明清楚

### 4. 交付檢查

- 文件中的命令與路徑可直接重現
- 新加入成員可依文件完成本機啟動與驗證
- 已知限制有明確記錄，不讓使用者誤判為 bug

## 驗證方式

- 由非原作者依 README 重新執行一次本機流程
- 檢查文件中所有命令、埠號與檔名是否與實際一致
- 檢查常見錯誤是否有對應排查方式

## 完成定義

- 文件足以支援開發、測試與基本交接
- 驗證步驟不依賴隱性知識或口頭補充
- 使用者可快速判斷問題出在前端、signaling 還是後端處理

## 風險與注意事項

- 若文件整理延後到最後，容易遺漏重要環境細節
- 若驗證只測 happy path，實際整合時會反覆踩錯誤情境
- 若命令與設定沒有同步更新，文件會很快失效
